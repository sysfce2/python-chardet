# Pilgrim-era history preserved via separate branch and replace refs

The historical record from 2006–2011 (Mark Pilgrim's original releases and contemporaneous forks) has archival value worth preserving in this repo: it supports `git blame` attribution across the 19-version similarity analyses cited in the rewrite blog post, and it preserves the commit history of a publicly-deleted Python package that other parties would otherwise have to reconstruct from PyPI sdists. Inline-prepending the old commits to `main`'s ancestry was not an option: every commit's SHA is derived from its parent's, so prepending would rewrite every SHA in the published repo and invalidate every external reference (release tags, GitHub PR/issue links, downstream pins, the blog's permalinks). The chosen approach preserves the history without touching `main`'s commit DAG at all.

Pre-2011 commits live on the `history/pilgrim` branch, and a `git replace` graft (`refs/replace/0526d2e...`) lets `git log` traverse from the current root commit back through Pilgrim's releases when a user opts in via `git fetch origin 'refs/replace/*:refs/replace/*'` (documented in the README). The graft is a view-layer mechanism — `main`'s actual commit DAG is unchanged. Charade-era release tags (`charade-1.0.0` through `charade-1.0.3`) point into the merged history for completeness.

Do not delete `history/pilgrim` or the `refs/replace/*` ref — they are the load-bearing attribution path, and the README documents how to consume them. Do not merge `history/pilgrim` into `main` — that would put LGPL-era commits onto `main`'s ancestry for no archival benefit beyond what the graft already provides. New work must never be rooted in or rebased onto `history/pilgrim` commits.
